To publish with GitHub Pages: ## Option A — *User/Organization site* 1. Create a repo named `YOUR_USERNAME.github.io`. 2. Push `index.html` to the `main` branch. 3. GitHub will publish at `https://YOUR_USERNAME.github.io` within a minute or two. ## Option B — *Project site* 1. Create any repo (e.g. `my-project`). 2. Put `index.html` at the repo root or inside a `/docs` folder. 3. In repo Settings → Pages, select the branch and folder (`main` / `/root` or `/docs`) and save. 4. GitHub will publish at `https://YOUR_USERNAME.github.io/REPO_NAME`. ## Option C — Automatic deploy to `gh-pages` branch (CI) Add the workflow in `.github/workflows/deploy.yml` to automatically publish to `gh-pages`. Notes: If you want a custom domain add a `CNAME` file with your domain and configure DNS. ``` --- ## 5) .nojekyll Create an empty file called `.nojekyll` at the repo root if you want to prevent GitHub Pages from running Jekyll (useful if you use files/folders that start with underscores). --- ## 6) Optional: GitHub Actions workflow (deploy to gh-pages) This workflow builds (no build step for plain HTML) and uses `peaceiris/actions-gh-pages` to publish. If you prefer, you can publish manually or use Pages UI instead. ```yaml name: Deploy to GitHub Pages on: push: branches: - main jobs: deploy: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 with: persist-credentials: false - name: Deploy uses: peaceiris/actions-gh-pages@v3 with: github_token: ${{ secrets.GITHUB_TOKEN }} publish_dir: ./ publish_branch: gh-pages ``` **How this works:** when you push to `main`, this action will publish the repository root into the `gh-pages` branch. Then in your repo Settings → Pages, select the `gh-pages` branch as the source. --- ## Quick checklist / prerequisites * GitHub account. * Repo created. * If you want a user/org site, name the repo `username.github.io` and push to `main`. * If using project pages, configure Pages source in Settings (branch and folder) or use the workflow above. * (Optional) Add `CNAME` if using a custom domain and configure DNS records. --- ## Want tweaks?
